Introducing

什么是 Test Agents？

Playwright 官方推出的 AI 测试系统。三个专门干测试的 Agent，形成完整的测试自动化链路。

以前是你写代码让机器跑，现在是你说需求让 AI 自己来

Three Agents · 各司其职
04 / 12
三个 Agent，一条链路

Planner → Generator → Healer

Planner
探索应用，产出 Markdown 格式的测试计划。可从零开始，也可基于 seed 测试扩展。
Input: 任务描述
Output: specs/*.md
Generator
把测试计划转成可执行的 Playwright 测试代码，实时验证选择器有效、断言合理。
Input: specs/*.md
Output: tests/*.spec.ts
Healer
跑失败的测试自动找原因、自动打补丁修好。Guardrails 机制避免无限重试。
Input: 失败的测试
Output: 修复后的代码

Agent 03 · 修复者

测试挂了？自动修

三个 Agent 里最省心的一个。跑失败的测试交给他，自动回放、诊断、打补丁。

01

重新回放失败的测试步骤

02

查看当前 UI，找到对应元素或流程

03

建议补丁方案（locator 更新、wait 调整、data 修复）

04

重新跑测试，直到通过或 guardrails 停止循环

"选择器大范围改动、功能逻辑变了、接口 schema 变了——这些靠 Healer 修不了。它处理的是'能跑但跑失败了'的情况。"

Quick Start · 快速开始
09 / 12
Quick Start

一条命令初始化

之后在 AI 编码工具里直接用。三个 Agent 可灵活组合，按需调用即可。

npx playwright init-agents
--loop=claude   # Claude Code 模式
--loop=opencode   # OpenCode 模式

生成的目录结构：specs/（测试计划）+ tests/（测试代码）+ .github/（agent 定义）+ playwright.config.ts。计划与测试一一对应，出了问题方便追溯。

Before / After · 三种模式对比

MCP vs CLI vs Test Agents

Before · 传统模式

MCP / CLI

  • MCP：给 AI 装了"眼睛"，你指挥，AI 执行
  • CLI：给 AI 装了"手脚"，AI 自主执行命令
  • MCP 写单个测试，CLI 搭自动化流程
  • 全程在场或完全不需要介入
After · Test Agents

自主 AI 系统

  • AI 自主完成目标，关键节点审核即可
  • 目标式，自主规划，模块级/项目级覆盖
  • Token 消耗适中（有中间产物可复用）
  • 写单个测试找 MCP，建整体覆盖上 Agents

Best Practices · 注意事项
11 / 12
实战避坑指南

四个关键注意事项

Tip 01
Seed要写好
整个 Agent 流程的基础。应用需要登录但 seed 没处理，Planner 探索时会撞墙
Tip 02
版本要同步
每次升级 Playwright 后重新跑 init-agents，定义文件不会自动同步
Tip 03
Healer非万能
功能逻辑变了、接口 schema 变了——这些需要人工介入，Healer 处理"能跑但失败"的情况
Tip 04
代码要过眼
断言力度、边界情况、测试间依赖——AI 生成不等于免审查
